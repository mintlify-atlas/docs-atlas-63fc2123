## Get Started in 5 Minutes

This guide will walk you through installing and running the WhatsApp Audio Transcriber bot, from setup to transcribing your first voice message.

<Steps>
  <Step title="Install FFmpeg">
    FFmpeg is required for audio format conversion. Install it using your system's package manager:

    <CodeGroup>
    ```bash macOS (Homebrew)
    brew install ffmpeg
    ```

    ```bash Ubuntu/Debian
    sudo apt update
    sudo apt install ffmpeg
    ```

    ```bash Windows (Chocolatey)
    choco install ffmpeg
    ```

    ```bash Arch Linux
    sudo pacman -S ffmpeg
    ```
    </CodeGroup>

    <Tip>
      Verify FFmpeg installation by running `ffmpeg -version` in your terminal.
    </Tip>
  </Step>

  <Step title="Clone and Install Dependencies">
    Clone the repository and install the required npm packages:

    ```bash
    git clone https://github.com/lucaboy/whatsapp-audio-transcriber.git
    cd whatsapp-audio-transcriber
    npm install
    ```

    This will install all dependencies including:
    - `whatsapp-web.js` - WhatsApp Web client
    - `smart-whisper` - Whisper.cpp wrapper
    - `fluent-ffmpeg` - FFmpeg wrapper for Node.js
    - `express` - Web server for authentication
  </Step>

  <Step title="Download Whisper Model">
    Download a pre-converted Whisper model in ggml format from [Hugging Face](https://huggingface.co/ggerganov/whisper.cpp).

    <Info>
      **Model Sizes**: Choose based on your accuracy and speed requirements:
      - `tiny` (~75MB) - Fastest, lower accuracy
      - `base` (~142MB) - Good balance for testing
      - `small` (~466MB) - Better accuracy
      - `medium` (~1.5GB) - High accuracy (recommended)
      - `large-v3` (~3.1GB) - Best accuracy, slower
    </Info>

    Example download commands:

    <CodeGroup>
    ```bash Download Medium Model (Recommended)
    # Create models directory
    mkdir -p models
    
    # Download medium model
    wget https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-medium.bin -O models/ggml-medium.bin
    ```

    ```bash Download Large V3 Model
    # Create models directory
    mkdir -p models
    
    # Download large-v3 model
    wget https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-large-v3.bin -O models/ggml-large-v3.bin
    ```

    ```bash Download Base Model (Fastest)
    # Create models directory
    mkdir -p models
    
    # Download base model
    wget https://huggingface.co/ggerganov/whisper.cpp/resolve/main/ggml-base.bin -O models/ggml-base.bin
    ```
    </CodeGroup>

    <Warning>
      Make note of the exact path where you save the model file - you'll need it in the next step.
    </Warning>
  </Step>

  <Step title="Configure Environment Variables">
    Copy the example environment file and configure your settings:

    ```bash
    cp .env.example .env
    ```

    Edit `.env` with your preferred settings:

    ```bash .env
    PORT=8080
    TRANSCRIPTION_REACTION='🤖'
    USE_GPU=false
    WHISPER_LOCAL_MODEL_PATH=models/ggml-medium.bin
    TRANSCRIPTION_LANGUAGE=auto
    ```

    <AccordionGroup>
      <Accordion title="Configuration Options">
        - `PORT`: Port for the authentication web server (default: `8080`)
        - `TRANSCRIPTION_REACTION`: Emoji to trigger transcription (default: `🤖`)
        - `USE_GPU`: Enable GPU acceleration for faster processing (default: `false`)
        - `WHISPER_LOCAL_MODEL_PATH`: Path to your downloaded Whisper model
        - `TRANSCRIPTION_LANGUAGE`: Language code or `auto` for automatic detection
      </Accordion>

      <Accordion title="Language Codes">
        Set `TRANSCRIPTION_LANGUAGE` to a specific language for better accuracy:
        - `en` - English
        - `es` - Spanish
        - `fr` - French
        - `de` - German
        - `it` - Italian
        - `pt` - Portuguese
        - `zh` - Chinese
        - `ja` - Japanese
        - `auto` - Automatic detection (default)
      </Accordion>

      <Accordion title="GPU Acceleration">
        If you have a CUDA-compatible GPU, set `USE_GPU=true` for faster transcription:
        
        ```bash .env
        USE_GPU=true
        ```
        
        <Note>GPU support requires additional setup. See the whisper.cpp documentation for details.</Note>
      </Accordion>
    </AccordionGroup>
  </Step>

  <Step title="Build and Start the Bot">
    Compile the TypeScript code and start the application:

    ```bash
    npm run build
    npm start
    ```

    You should see output indicating the server has started:

    ```
    Server listening on port 8080
    ```

    <Tip>
      For development, you can use `npm run dev` to run with hot-reload using nodemon.
    </Tip>
  </Step>

  <Step title="Authenticate with WhatsApp">
    Open your browser and navigate to:

    ```
    http://localhost:8080
    ```

    You'll see a QR code displayed. Scan it with your WhatsApp mobile app:

    <Steps>
      <Step title="Open WhatsApp on your phone">
        Launch the WhatsApp mobile app.
      </Step>
      <Step title="Access WhatsApp Web">
        Tap the three dots (⋮) menu → **Linked Devices** → **Link a Device**
      </Step>
      <Step title="Scan the QR code">
        Point your phone camera at the QR code on your computer screen.
      </Step>
    </Steps>

    Once authenticated, the page will display:

    ```
    Connected. You can close this page.
    ```

    <Info>
      The bot uses `LocalAuth` strategy, so you won't need to scan the QR code again on subsequent runs (unless you delete the `.wwebjs_auth` folder).
    </Info>
  </Step>

  <Step title="Test the Transcription">
    Now let's test the bot with a voice message:

    1. **Send yourself a voice message** in WhatsApp (or ask a friend to send you one)
    2. **React to the voice message** with the 🤖 emoji
    3. **Wait a few seconds** for the transcription to process
    4. **Receive the transcription** as a reply to the voice message

    The bot will reply with:

    ```
    [WA-TRANSCRIBER-BOT] Your transcribed text will appear here
    ```

    <Warning>
      The reaction must be from you (the account running the bot) and within the last 2 minutes. Older reactions are ignored to prevent processing old messages on startup.
    </Warning>
  </Step>
</Steps>

## How It Works

Here's what happens behind the scenes when you react to a voice message:

```typescript
client.on('message_reaction', async reaction => {
    // Only process reactions from yourself with the configured emoji
    if (
        !reaction.id.fromMe || 
        reaction.reaction !== transcriptionReaction || 
        moment.unix(reaction.timestamp).isBefore(moment().subtract(2, 'minutes'))
    ) {
        return;
    }

    // Get the message that was reacted to
    const message = await client.getMessageById(reaction.msgId._serialized);

    if (whisper && message?.type === 'ptt') {
        // Download the voice message
        const audio = await message.downloadMedia();
        
        // Convert OGG to WAV format
        const filePathWithoutExtension = `./tmp/audio/${Date.now()}`;
        fs.writeFileSync(`${filePathWithoutExtension}.ogg`, audio.data, 'base64');
        
        ffmpeg(`${filePathWithoutExtension}.ogg`)
            .audioFrequency(16000)
            .toFormat('wav')
            .save(`${filePathWithoutExtension}.wav`)
            .on('end', async () => {
                // Transcribe using Whisper
                const { channelData } = decode(fs.readFileSync(`${filePathWithoutExtension}.wav`));
                const task = await whisper.transcribe(pcm, { language: transcriptionLanguage });
                const transcription = (await task.result).map((result) => result.text).join(' ');
                
                // Reply with transcription
                if (transcription) {
                    message.reply(`[WA-TRANSCRIBER-BOT] ${transcription}`);
                }
            });
    }
});
```

## Troubleshooting

<AccordionGroup>
  <Accordion title="QR Code Not Appearing">
    - Check that port 8080 is not already in use
    - Try changing the `PORT` in your `.env` file
    - Check the console for any error messages
    - Ensure Puppeteer dependencies are installed (may require additional packages on Linux)
  </Accordion>

  <Accordion title="Transcription Not Working">
    - Verify the Whisper model path in `.env` is correct
    - Ensure the model file exists and is not corrupted
    - Check that FFmpeg is installed: `ffmpeg -version`
    - Make sure you're reacting from the same account running the bot
    - Only voice messages (PTT) are supported, not regular audio files
  </Accordion>

  <Accordion title="Bot Disconnects Frequently">
    - This may happen if WhatsApp Web detects unusual activity
    - Try using the `webVersionCache` configuration (already included in the code)
    - Ensure stable internet connection
    - Check WhatsApp Web status on other devices
  </Accordion>

  <Accordion title="Slow Transcription">
    - Use a smaller model (`base` or `small` instead of `large-v3`)
    - Enable GPU acceleration if you have a compatible GPU: `USE_GPU=true`
    - Ensure your system has sufficient RAM (at least 4GB free)
    - Close other resource-intensive applications
  </Accordion>
</AccordionGroup>
